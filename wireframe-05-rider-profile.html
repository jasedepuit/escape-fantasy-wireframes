Screen 5 of 5
Rider Profile
A bottom sheet modal triggered by tapping any rider row in the Rider Selection screen. Lets the user research a rider before committing to a pick — no navigation away from the selection flow.
Modal
1
Drag handle — swipe down or tap outside the modal to dismiss and return to the Rider Selection screen.
2
Tapped row highlight — the rider row in the background stays visually selected (orange tint) so the user keeps context of which rider they're viewing.
Rider Identity
3
Photo, name, team, flag — rider photo placeholder (real photo in production). Nationality flag + team shown below the name.
4
Speciality badge — GC, Climber, Sprinter, Puncheur etc. Helps users quickly understand a rider's strengths relative to the upcoming stage type.
5
Bio strip — Age, Height, Weight. Compact three-column layout so it doesn't take up much vertical space.
Performance Data
6
Season stats — Points (highlighted in orange as the fantasy-relevant metric), Wins, Podiums, Top 10s for the current season.
7
Recent results — Last 5 results with race name, date, and fantasy points earned. Position circle: solid orange = win, outline orange = top 3, grey = other.
Action
8
Pick CTA — confirms the rider pick for the current stage and dismisses the modal. Label reads "Pick [Name] for Stage [N]" so the user is always clear what they're confirming. Button is disabled (grey) if the rider is already used or the stage is locked.
Modal sits on top of Screen 4 (Rider Selection). The underlying nav bar and confirm bar remain visible behind the dim overlay but are non-interactive while the modal is open.
