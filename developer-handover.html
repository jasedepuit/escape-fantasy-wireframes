2 · Authentication & Sessions

Ghost CMS Single Sign-On

Escape Fantasy uses Escape Collective's Ghost membership system as the sole auth provider. Users sign in with their EC email address — no separate Escape Fantasy password is ever created or stored. A free EC account is sufficient; paid membership is not required to play.

Key Decision — Auth method

Use Ghost's authentication code (OTP) flow, not magic links. Magic links cannot reliably deep-link back into a native app after the user taps the link in their email client. Auth codes are entered directly inside the app — no browser handoff required.

OTP Flow

  • User enters EC email address
  • Tap "Email me a code" → Ghost sends 6-digit OTP
  • User enters code in-app; code expires in 10 minutes
  • Correct code → Ghost returns auth token
  • "Resend code" has a 60-second cooldown

Session Management

  • Store Ghost auth token in secure device storage
  • Returning users with valid token skip login entirely
  • Login screen only shown when no valid session exists
  • Logout: confirm sheet → clear token → return to Login
  • Token refresh strategy: follow Ghost API docs

Onboarding Flag

The three onboarding screens (22–24) are shown exactly once, immediately after the first successful login. A boolean flag (e.g. ef_onboarding_complete) must be persisted in local storage or equivalent. Once set, onboarding never triggers again — even after logout and re-login on the same device. A Skip button is available on all three screens; tapping it sets the flag and navigates directly to Home.

EC Account Creation

The Login screen offers a "Create a free EC account" path that opens escapecollective.com/join in the device's default browser. If the OS supports universal links / app links, implement a deep-link back to the app's login screen after account creation. If not, the user returns manually and signs in.

3 · Competition Formats

Every competition in Escape Fantasy is one of two formats. The format determines which screens a user sees after entering a competition from the Competition Browser or Home screen.

Stage Race

  • One rider pick per stage
  • Picks earn points from real stage results
  • Highest cumulative points across all stages wins
  • Picks lock at a per-stage deadline
  • Auto-pick available if user misses deadline

Screens: My Picks (03) → Rider Selection (04) → Rider Profile (05) → Per-Stage Results (07)

Fantasy Teams

  • Pick a squad of 9 riders across 4 classes + 1 wildcard
  • Spend a credit budget (amount varies per event)
  • All 9 riders score every stage
  • Transfer windows allow squad changes at rest days
  • No automatic substitutions ever

Screens: Team Selection (19) → Team Overview (20)

Key Decision — Format branching point

The format branch happens at the Competition Browser (02) and Home card CTAs (01). Stage Race competitions show a "My Picks →" button; Fantasy Teams competitions show a "View Team →" (or "Enter Team →" if not yet entered). Screens 03–07 are Stage Race only. Screens 19–20 are Fantasy Teams only. The Leaderboard (08) and League screens (09–14) are shared across both formats.

4 · Fantasy Teams Rules

Squad Structure

Every Fantasy Teams squad has exactly 9 slots across 5 types. Slot counts may be configurable per competition by the competition director, but the default is below.

Budget System

  • Credit budget is set per competition (varies event to event)
  • Rider prices set by the competition director, reflecting form
  • Budget bar updates live as riders are added/removed
  • Budget bar label turns red below 10 credits remaining
  • Over-budget riders: dimmed, price shown in red, + disabled
  • Confirm button disabled until all 9 slots filled and within budget
  • Unlimited changes allowed until the entry deadline

Wildcard Slot

  • Accepts a rider from any class
  • If a user taps + on a rider whose class slots are full, offer to place them in the Wildcard slot with a confirmation prompt

Transfer Windows

  • Transfer windows occur at intervals set by the competition director — typically at rest days
  • Window open/closed state shown by a badge in Team Overview header
  • When open: Transfer CTA turns orange, "Make Transfer →" is active
  • When closed: CTA shows next window countdown (greyed out)
  • No automatic substitutions, ever

DNF / Abandoned Riders

Riders who abandon or are disqualified are shown with a red DNF badge and dimmed to ~45% opacity in Team Overview. Their accumulated points up to the disqualification deadline stand. DNF state is a strong signal to act during the next transfer window. When a window is open, consider adding a direct "Replace →" shortcut on DNF rows in addition to the standard Transfer CTA.

5 · Notification Architecture
Key Decision — Bell icon placement

The notification bell icon lives on the Home screen only (top-right header). It does not appear on Competition Browser, My Picks, Rider Selection, or any other screen. This keeps the notification entry point unambiguous and prevents UI clutter on task-focused screens.

Key Decision — Nav badge placement

The unread-notification badge dot lives on the Home tab in the bottom nav (not the Account tab). This is the standard iOS/Android convention — the badge lives on the tab that owns the content. The Account tab has no badge at any time.

Notification Feed (Screen 18)

  • Accessed by tapping the bell icon on Home only
  • Home tab remains active in bottom nav (child of Home, not a separate tab)
  • Notifications persist for a rolling window (e.g. 30 days)
  • Not user-deletable — no dismiss/swipe-to-delete
  • Individual rows marked read on tap; "Mark all read" bulk action
  • "Mark all read" button only visible when unread items exist
  • Time-sensitive notifications (picks lock warnings) float to the top regardless of chronological order
  • Tapping any notification deep-links to the relevant screen

Notification Types

  • Picks closing soon — highest priority; navigates to My Picks for that stage
  • Rank change — ↑ green / ↓ red; navigates to relevant leaderboard; rank-down is off by default
  • League enrolled — director enrolled your league into a new competition; navigates to that competition
  • Chat mention — league chat @mention; navigates to League Detail chat tab

Preferences Shortcut

The ⚙ icon in the Notification Feed header navigates directly to Notification Preferences (Screen 17). This is intentional — users reaching for mute controls are in context here.

Notification Preferences (Screen 17)

Grouped into three sections: Fantasy (picks closing soon, leaderboard rank change, league rank change — all default on), Leagues (new league member, competition enrolled — both default on), and Chat (new messages — default on; mentions-only subordinate toggle — default off). Chat notifications are per-app, not per-league. An OS permission banner is shown when system notifications have not been granted, with a deep-link to device settings. Preference toggles are visible and saveable regardless of OS permission state.

7 · Third-Party Integrations

Ghost CMS — Auth & Membership

Used for authentication (see §2) and to validate EC membership status. The Account screen shows an "EC Member" badge on the profile header — this should reflect actual subscription status from Ghost. Access to News tab content may be membership-gated depending on EC's article paywall settings; confirm with the EC team.

  • Auth endpoint: Ghost's members/api — use the auth-code (OTP) flow
  • Session token storage: device secure storage (Keychain / Keystore)
  • Free account sufficient; no paid membership gate on game features

EC RSS / Ghost API — News Feed

The News tab (Screen 15) surfaces Escape Collective articles. Use the Ghost Content API or RSS feed. Articles should open in an in-app browser or the device's default browser — confirm the preferred behaviour with EC. Stage preview articles that can link to a pick flow require the article to carry metadata identifying the relevant stage/competition.

Race Results Data

Scoring in both formats depends on real-time race results. The wireframes do not specify the results data source — this needs to be confirmed with the EC team. Key requirements implied by the UI: stage finish positions, GC standings, sprint classification points, and KOM points per stage; rider abandonment / disqualification status; results available promptly after each stage to trigger scoring and notifications.

8 · Design Tokens

All wireframes use the following CSS custom property system. Copy this block as the foundation for the production design token file.

/* ── Escape Fantasy — Core Design Tokens ── */ :root { /* Brand */ --accent: #FF6F42; /* primary orange — CTAs, active states, badges */ --accent-lo: rgba(255,111,66,0.12); /* Dark mode (default) */ --bg-screen: #111; /* outermost background */ --bg-card: #1C1C1E; /* card / list row backgrounds */ --bg-input: #242424; /* text inputs, chips */ --border: rgba(255,255,255,0.08); --text-primary: #E8E8E8; --text-muted: #999; --text-dim: #555; /* Rider class colours */ --class-ar: #5B8BF5; /* All-Rounder — blue */ --class-cl: #34C759; /* Climber — green */ --class-sp: #FF9500; /* Sprinter — orange */ --class-uc: #AF52DE; /* Unclassed — purple */ --class-wc: #FF6F42; /* Wildcard — accent (same as --accent) */ /* Semantic */ --green: #34C759; /* rank up, transfer window open */ --red: #FF3B30; /* rank down, DNF, over-budget, destructive actions */ --blue: #5B8BF5; /* Fantasy Teams format badge */ } /* Light mode overrides — applied via body.light-mode */ body.light-mode { --bg-screen: #F2F3F5; --bg-card: #FFFFFF; --bg-input: #F0F0F0; --border: rgba(0,0,0,0.09); --text-primary: #111; --text-muted: #555; --text-dim: #999; }

Theme Behaviour

  • Default on first launch: follow device system preference
  • User can override in Account → Settings (segmented Light / Dark control — no sub-screen)
  • Choice persists across sessions
  • Competition card images and overlays: overlay text (.race-name, stage badges) must always be color: #fff — hardcoded, not themed — because they sit over a photo gradient
  • App header bell icon: must switch from white (dark mode) to #111 (light mode)
9 · Account & Settings Notes

Profile

  • Shows avatar initials, display name, email, EC membership badge
  • Tapping navigates to a profile edit screen (not yet wireframed)
  • EC Member badge reflects Ghost subscription status

Log Out

  • Label and icon shown in red (destructive action signalling)
  • Must trigger a confirmation sheet before logging out
  • Tapping away from the sheet cancels
  • On confirm: clear session token → return to Login (Screen 21)

Version Footer

App version string in the Account screen footer should be pulled from build metadata at compile time. Terms of Service and Privacy Policy links open in the device's default browser.
