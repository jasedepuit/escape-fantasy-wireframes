Screen 21 · Login — Annotations
① EC Account — Single Sign-On
Authentication uses Escape Collective's Ghost membership system. The user signs in with the email address linked to their EC account — no separate Escape Fantasy password is required. EC members who are already logged in on the device may be auto-signed in via a saved session token, bypassing this screen entirely.
② Authentication Code
Tapping "Email me a code" triggers Ghost's auth-code flow — a 6-digit OTP is emailed to the entered address. The code is single-use and expires in 10 minutes. On tap, the form collapses and the code-entry state (④) appears. This avoids magic links, which cannot deep-link back into the app reliably.
③ Create EC Account
Opens escapecollective.com/join in the user's default browser (same as article links). A free-tier EC account is sufficient to play — no paid membership required. After creating the account the user returns to the app and signs in normally. Dev note: deep-link back to app after account creation if the OS allows it.
④ Auth Code Entry State
Replaces the form after submission. Shows the destination email for confirmation. User enters the 6-digit code directly in the app. "Resend code" re-triggers the Ghost auth-code API with a 60-second cooldown. On correct code entry the app receives the auth token and advances to Onboarding 1 (first time) or the Home screen (returning users). No browser redirect needed.
Returning Users
This screen is only shown when no valid session exists. Returning users with an active token skip directly to the Home screen. The onboarding flow (Screens 22–24) is shown once only after first successful login — controlled by a persisted flag in local storage.
