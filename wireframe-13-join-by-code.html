Screen 13 — Join by Code
Code Format
6-character alphanumeric code, case-insensitive. Rendered uppercase in the boxes. The hidden input captures keyboard entry; the visual boxes render each character. On mobile, tapping the boxes triggers the native keyboard.
Auto-Lookup
Lookup fires automatically when all 6 characters are entered — no separate search tap needed. Valid code → league preview card appears, Join button enables. Invalid code → error banner + shake animation on the boxes. Editing any character resets to the blank state.

In this wireframe: TDF026 resolves to "Tour de France Fanatics". Any other 6-char string triggers the error state.
League Preview Card
Shown only after a valid lookup. Displays league name, logo/colour, member count, and director name — enough context to confirm the user is joining the right league before committing. Competition/stage fields removed: a league can be active in multiple competitions simultaneously so those values are ambiguous here.
Join Button
Disabled (35% opacity) until a valid lookup succeeds. On tap: transitions to "Joined! 🎉" with a green background, then navigates to League Detail (Screen 10). If the league is already at capacity or the user is already a member, the button shows an appropriate error state instead.
Entry Points
• League List (Screen 09) — "Join by Code" outline button in the action row.

• Deep link — a shared invite URL can pre-fill the code and trigger lookup automatically, skipping the manual entry step.
