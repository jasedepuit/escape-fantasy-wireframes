Screen 7
Per Stage Results
Accessed by tapping any player's name on the Leaderboard. Shows their pick for every stage alongside the actual stage winner, so the viewer can understand how their score was built up across the race.
Header
1
Player name + nationality flag — identifies whose picks are being viewed. Works for viewing your own results and scouting other players in your league.
2
Competition context — sub-label shows the race name and format so the user always knows which competition they're viewing results for.
Stage Rows
3
Pick row (*) — the player's selected rider with their nationality flag. Points scored shown right-aligned in orange. The asterisk visually separates the pick from the winner row below.
4
Winner row (🥇) — the actual stage winner shown directly below the pick for easy comparison. Where available, a time gap is shown (e.g. "+12s") for context on how the stage was won.
5
Pick = winner highlight — when a player's pick won the stage, their pick row is highlighted in orange (see Stage 4). The winner row still appears below so the pattern stays consistent.
6
No pick — future or skipped stages show a single muted row. No winner is shown for future stages as the race hasn't happened yet.
Footer
7
Total — sum of all points scored to date. Matches the player's score on the Leaderboard, giving the user confirmation they're seeing the right data.
Entry point: Leaderboard only. The My Picks screen (Screen 3) already shows the current user's per-stage points, so this screen is not linked from there. When viewing your own results, the back button returns to the Leaderboard rather than My Picks.
