Stage 15 is the queen stage of this year's Tour de France, featuring the legendary ascent to Alpe d'Huez as its final climb. Riders face 198km from Grenoble through the Chartreuse massif before tackling the iconic 21 hairpins.

The decisive climb begins at kilometre 180. The Col de la Croix de Fer precedes it, ensuring legs will already be tested well before the final showdown. GC riders will need to time their efforts carefully.

Expect attacks from the pure climbers early on the Alpe, with GC leaders trying to control the race until the final 3km where the gradient eases before a short steep finish.

Stage Header — Profile Image
The elevation profile graphic gives riders immediate visual context for the stage character — they can see at a glance whether it's a flat sprint, a mountain finish, or a punchy day. The summit marker and distance callouts are sourced from First Cycling data. In production this is a real profile image from the race organisers.
②
Cut-off Badge in Header
The live cut-off countdown is moved into the app header as a compact badge — freeing up screen space compared to a full-width strip. Pulsing red dot maintains urgency. Hidden on future stages where no live window is open.
③
Overview / Description Tabs
Overview tab (default): shows stage stats + the full rider picker. Description tab: replaces the lower half with a scrollable stage narrative — useful for users who want context before picking. Switching back to Overview restores the rider list at the same scroll position. The rider list is intentionally hidden in Description mode to avoid confusion.
④
Stage Stats Row
Three key facts: Distance, Type, Finish. Mirrors the column layout used in the existing app. Tapping any stat cell does nothing — these are read-only. The Type value ("Mountain", "Flat", "ITT" etc.) corresponds to the stage classification from the race organiser.
⑤
AI Suggestion Banner
Collapsed to a single line — sparkle icon, "AI Pick" label, recommended rider name. Tapping "View →" opens the Rider Profile (Screen 5) for that rider. This keeps the AI recommendation visible and accessible without dominating the list. The suggested rider also appears in the available list below so the user can compare before deciding.
⑥
Rider Rows — Tap Behaviour
Tapping the rider name/team area opens Rider Profile (Screen 5) as a modal, showing full stats before committing. Tapping "Pick →" directly selects the rider without visiting the profile — entering the selected state. Both paths ultimately reach the same confirm bar.
⑦
Selected State + Confirm Bar
Selected rider row gets an orange left border and tinted background. "Pick →" becomes "✓ Selected". A solid orange confirm bar appears above the bottom nav — tapping it saves the pick and returns to My Picks. The confirm bar persists even when the user scrolls or searches, ensuring the action is always reachable.
⑧
Used This Race
Riders already assigned to a previous stage are shown dimmed at 40% opacity with a "Used · St.X" label. They cannot be selected — tapping has no effect. Shown at the bottom of the list rather than hidden entirely, so users know these riders exist but are unavailable. This is a core stage race rule: each rider can only be used once.
