Screen 19 · Team Selection — Annotations
① Screen Context
Shown for Teams-based competitions only — not used for Stage Race format (which uses Rider Selection, Screen 04). Accessed via Enter Team → on the Competition Browser, or Make Transfer → when a transfer window is open. When arriving mid-competition during a transfer window, slots are pre-filled with the current team.
② Entry Deadline
Countdown to team lock (entry deadline). Once expired the screen becomes read-only and the Confirm button is replaced with a locked state. Per the rules: unlimited changes are allowed up until this deadline. During a transfer window, this shows the window close time instead.
③ Budget Bar
Credit total is set per competition (varies event to event). Bar fills as credits are spent and updates live with each add/remove. Remaining label turns red when under 10 credits remaining. Rider prices reflect past results and recent form, as set by the competition director.
④ Squad Strip
Horizontal scroll strip showing all 9 slots: 2× All-Rounder · 2× Climber · 2× Sprinter · 2× Unclassed · 1× Wildcard. Slot counts are configurable per competition. Filled slots show class badge + surname + price + × to remove. Empty slots show class badge + +. Tapping a filled slot removes that rider and scrolls the pool to their row.
⑤ Wildcard Slot
The ★ Wildcard accepts a rider from any class, letting managers specialise. When tapping + on a rider whose class slots are already full, the app offers to place them into the Wildcard slot instead (with a confirmation prompt).
⑥ Class Filter Tabs
Filter the rider pool by the four classes from the rules: All-Rounder · Climber · Sprinter · Unclassed. The "All" tab shows every rider. Left-border colour on each row matches the class for quick scanning.
⑦ Sort Control
Default sort: Price ▾ (highest first, as star riders are most decision-worthy). Also sortable by name A–Z or by recent form. Sort persists when switching filter tabs.
⑧ Over-Budget State
Riders whose price exceeds remaining credits are dimmed and their + button is disabled. Price displays in red to signal the constraint. This updates live — removing a selected rider may re-enable over-budget options.
⑨ Team Name
A compact inline field between the header and budget bar. Required before the team can be confirmed — the Confirm button should remain hidden if no name has been entered. The team name is visible to other league members. Default placeholder: e.g. Chaingang Heroes.
Confirm Button — conditional visibility
Hidden entirely while the team is incomplete — do not show a greyed-out button. The Confirm button only appears (orange, full-width) once all slots are filled and a team name has been entered. Button label echoes the team name: "Confirm Team — [Name] →". On confirm, team is saved and the user is taken to the Team Overview screen. Can be re-submitted any number of times before the deadline. See Phone 2 for the complete state.
