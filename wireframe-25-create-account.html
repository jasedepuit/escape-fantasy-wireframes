Screen 25 · Create Account — Annotations
Form State
①
Display name — Free text, 2–40 chars. This becomes the user's visible identity on leaderboards and in leagues. Stored as name on the Ghost Member record. Shown pre-populated with email prefix if possible.
②
Email address — Must match an active Escape Collective membership. On submit, the backend checks the Ghost Members API for a matching member. If not found: show inline error "No Escape Collective membership found for this email." If found but already has account: show "An account already exists — log in instead."
③
Create Account button — On tap: (1) validate fields client-side, (2) POST to Ghost Members API (undocumented) to create the member record with display name + email, (3) trigger OTP email via the same mechanism as Login, (4) transition to OTP state. Button shows loading state during API call.
OTP State
④
OTP verification — Identical behaviour to the Login OTP screen. iOS AutoFill should detect the code from the email. 6-digit numeric input, single combined field internally. Back arrow returns to form (does not delete the created account).
⑤
Verify & Continue — On successful OTP validation, session is established. User is routed to Onboarding Screen 22 (not Home) since this is a first-time account creation. The onboarding flow collects country and home view preference before landing on Home.
Error States
• No membership found — Inline error below email field: "No Escape Collective membership found for this email."
• Account already exists — Inline error: "You already have an account. Log in instead →"
• Invalid OTP — Shake animation on boxes, error text: "Incorrect code — please try again."
• Expired OTP — "Code expired. " + Resend link.
• Network error — Toast: "Something went wrong. Please try again."
Flow
Login (21) → Create Account (25) → OTP verify → Onboarding 1 (22) → Onboarding 2 (23) → Onboarding 3 (24) → Home (01)
