Interaction Annotations
①
Competition switcher banner
The persistent context indicator. Tells the user which competition they're currently "inside" on the Compete tab. Tapping "Change ›" opens this Competition Browser screen. The selected competition becomes the active context for My Picks, Leaderboard, and Stage Detail sub-screens. On the Competition Browser itself, the banner shows the current selection so the user can see what they'd be changing.
②
"of X players" — participant count always visible
Participant count is shown on every joined competition card, directly alongside the user's rank. Never hidden. Resolves the top user survey complaint. Tapping the rank/players stat drills into the full leaderboard for that competition.
③
Team squad size indicator
Team Competition cards show "N riders / in squad" instead of a points-behind-leader stat. In the team format, squad depth is the relevant context for this view. The developer should note: squad size varies by competition (5-10 riders, admin-configured). Display as "N of 8 riders" if competition has a fixed squad size.
④
Late-entry cut-off for active competitions
Users can join some active competitions after they've started (admin-configured). When entry is still open, show the remaining entry window prominently in orange. When entry is closed (not joined), the Join button is replaced by a disabled "Entry closed" state (grey, non-tappable). This state is NOT shown in this wireframe but the developer must implement it.
⑤
Join CTA for active competitions
Tapping "Join Competition →" initiates the join flow. For Stage Race: user is immediately enrolled and taken to My Picks to make their first pick (if still within current stage cut-off) or see the schedule. For Team Competition: user is taken to the Team Selection screen to pick their squad before entry deadline. The join flow differs by format — the developer must branch on competition type.
⑥
Upcoming competition join
Pre-registration for an upcoming competition. Tapping "Join →" confirms registration immediately (no squad selection yet for team format — that opens separately). Badge changes to "JOINED ✓" and button disappears. The card then matches Card 5 state.
⑦
Team selection window — separate from entry deadline
For Team Competition format, there are TWO distinct deadlines: (a) entry registration deadline (you must join before this), and (b) team selection deadline (you must pick your squad before this). Both must be displayed. Entry can close before team selection opens (i.e., you register early but pick your team closer to race start). This is a critical developer note — two separate dates, two separate states.
⑧
Finished competitions section
Collapsed by default — shows only a count ("FINISHED (3)"). Tapping the section label expands it. Finished cards are fully desaturated and non-interactive except for "View Results →". The developer should implement the section as an expandable accordion. Final rank is always shown so the user can recall past performance.
❓ Open Questions (Section 9)
Q1 — Transfer mechanic for team format: Card 2 (2026 Grand Tour Season, active team comp) shows "8 riders / in squad" but does not show a transfer window alert on the competition browser card. If transfers are enabled, a small "Transfer window open" indicator should appear on the card. The exact mechanic (free transfers between races, or limited 1-2) determines whether this is a prominent CTA or a subtle badge. Needs decision before dev.
Q9 — League scoring override UX: Each competition card could show a small indicator if the user's league has overridden the scoring type (e.g. "Your league: Time-based scoring"). Where this surfaces — on the competition card vs. inside the competition detail — is unresolved.
